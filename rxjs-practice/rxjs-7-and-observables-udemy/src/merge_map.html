Observable - mergeMap

mergeMap is similar to switchMap, but it will not cancel the previous inner observable when a new value is emitted. mergeMap A is running, and mergeMap B is emitted, then mergeMap A will not be canceled and mergeMap B will be executed. subcriptions will be concurrent. Memory leaks hards to notice. No defined order of execution.
